================================================================================

For every Python package (directory with Python files), create __init__.py:

```bash

  [ -f {}/__init__.py ] || touch {}/__init__.py \;
```

STEP 4: Ask Claude Code to fill in files
================================================================================

For EACH file, use this prompt template:

---

Create: impact-analysis-system/core/types/service.py

Context:
  - Part of polyrepo impact analysis system
  - Python 3.10+, Pydantic v2, type hints required
  
Purpose:
  Define core data structures: Service, ExternalConnection, CodeArtifact, TestCase, Change

Required classes:

1. Service
   Fields: id (str), name (str), repoUrl (str), language (str), framework (str), owner (str), createdAt (datetime), lastUpdatedAt (datetime), isActive (bool)
   
2. ExternalConnection
   Fields: id (str), type (str), sourceServiceId (str), targetServiceId (str|None), targetName (str), protocol (str), endpoint (str), direction (str), frequency (float), criticality (str), contractStatus (str), dataFlow (dict), discoveredAt (datetime), lastObservedAt (datetime)
   
3. CodeArtifact
   Fields: id (str), repoId (str), type (str), name (str), file (str), lineRange (dict), isPublic (bool), version (str|None), externalConnections (list[str])
   
4. TestCase
   Fields: id (str), repoId (str), type (str), name (str), file (str), lineRange (dict), duration_ms (int), flakiness_score (float), priority (str), affectedRepos (list[str])
   
5. Change
   Fields: id (str), repoId (str), type (str), timestamp (datetime), description (str), files (list[dict]), affectedArtifacts (list[str])

Constraints:
  - Use Pydantic v2 BaseModel
  - All classes should be serializable to JSON
  - Include docstrings
  - Use type hints for all fields
  - Make classes immutable (frozen=True)

---
